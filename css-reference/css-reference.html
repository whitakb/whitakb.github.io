CSS Ruleset

Selectors

Universal Selectors

/* Selects all elements */
* {
    color: green;
}
  • * - matches all elements
  • Can be used with namespaces
    • ns|* - matches all elements in namespace ns
    • *|* - matches all elements
    • |* - matches all elements without any declared namespace

Type Selectors

Based on HTML element type

/* All <a> elements */
a {
    color: red;
}
  • a - matches all <a> elements
  • ns|a - matches all <a> elements in namespace ns
  • *|a - matches all <a> elements
  • |a - matches all <a> elements without any declared namespace

Class Selectors

Based on class attribute.

/* All elements with class="spacious" */
.spacious {
    margin: 2em;
}

/* All <li> elements with class="spacious" */
li.spacious {
    margin: 2em;
}

/* All <li> elements with class list that
   includes both "spacious" and "elegant" */
li.spacious.elegant {
    margin: 2em;
}

Equivalent to select by attribute: [class~=class_name]

ID Selectors

Matches an element with matching id (exactly). 

/* The element with id="demo" */
#demo {
    border: red 2px solid;
}

Attribute Selectors

Based on the presence or value of a given attribute. 

/* <a> elements with a title attribute */
a[title] {
    color: purple;
}

/* <a> elements with an href matching
   "https://example.org" */
a[href="https://example.org"] {
    color: green;
}

/* <a> elements with an href containing "example" */
a[href*="example"] {
    font-size: 2em;
}

Syntax

  • [attr] - elements with an attribute named attr
  • [attr=value] - elements with attribute named attr whose value matches value exactly
  • [attr~=value] - elements with attribute named attr whose value is a whitespace-separated list, one of which is exactly value
  • [attr|=value] - (generally used for language subcode matches) elements with attribute named attr whose value is exactly value or begins with value followed by a hyphen
  • [attr^=value] - elements with attribute named attr whose value is prefixed with value
  • [attr$=value] - elements with attribute named attr whose value is suffixed by value
  • [attr*=value] - elements with attribute named attr whose contains at least one occurance of value within the string
  • [attr ... i] - an i (or I) before the closing bracket causes a case-insensitive match
  • [attr ... s] - an s (or S) before the closing bracket causes a case-sensitive match

Selector List

Grouping

Selects all matching nodes

span,
div {
    border: red 2px solid;
}

If any selector in the list is invalid, then the entire list will be invalid.

Can make a "forgiving" list using :is(h1, h2:maybe-unsupported, h3). In this list, any valid selectors will still be valid. This has different specificity rules than just a list.

Relative Selector List

Can have a list which is parsed as relative selectors 

h2:has(+ p, + ul.red) {
    font-style: italic;
}

The example will be applied to any h2 heading that is immediately followed by <p> or <ul class="red">

Selectors: Combinators

Descendent Combinator

Select nodes that are decendents of the first element 

/* List items that are descendants 
of the "my-things" list */
ul.my-things li {
    margin: 2em;
}

The descendant combinator — typically a single space (" ") character — combines two selectors such that elements matched by the second selector are selected if they have an ancestor (parent, parent's parent, parent's parent's parent, etc.) element matching the first selector. Selectors that utilize a descendant combinator are called descendant selectors.

The descendant combinator is technically one or more CSS white space characters — the space character and/or one of four control characters: carriage return, form feed, new line, and tab characters — between two selectors in the absence of another combinator. Additionally, the white space characters of which the combinator is comprised may contain any number of CSS comments.

Child Combinator

Select nodes that are direct children of the first element 

/* List items that are children 
of the "my-things" list */
ul.my-things > li {
    margin: 2em;
}

The child combinator (>) is placed between two CSS selectors. It matches only those elements matched by the second selector that are the direct children of elements matched by the first.

This is stricter than the descendant combinator, which matches all elements matched by the second selector for which there exists an ancestor element matched by the first selector, regardless of the number of "hops" up the DOM.

General Sibling Combinator

Select nodes that are following the first element and are children of the same parent 

/* Paragraphs that are siblings of 
and subsequent to any image */
img ~ p {
    color: red;
}

Adjacent Sibling Combinator

Select nodes that are immediately following the first element and are children of the same parent 

/* Paragraphs that come immediately 
after any image */
img + p {
    color: red;
}

Column Combinator

Experimental and not really supported yet

/* Table cells that belong to 
the "selected" column */
col.selected||td {
    background: gray;
}

The column combinator (||) is placed between two CSS selectors. It matches only those elements matched by the second selector that belong to the column elements matched by the first.

Selectors: Pseudo-Classes and Pseudo-Elements

Pseudo-Classes

React to the state of an element

/* Any button over which the user's 
pointer is hovering */
button:hover {
    color: blue;
}

A pseudo-class is a keyword added to a selector that specifies a special state of the selected element(s), like :hover

Functional pseudo-classes :is() Takes a list of selectors as the argument, and selects any element that meets any of the selectors in the list. Has advantages over just using a list of selectors:
  • Allows for writing large list of selectors in compact form
  • Is a "forgiving" list, so if one of the selectors is invalid, the others will still work 

:is(section, article, aside, nav) h1  {
    font-size: 25px;
}

Pseudo elements are not valid in the selectors list.

Similar to :where, but has different specificity.

:not() Like :is, but the negation of the selector list. Has specificity impacts.
:where() Like :is, but with different specificity.
:has() Represents an element if any of the relative selectors that are passed as an argument match at least one element when anchored against this element. This pseudo-class presents a way of selecting a parent element or a previous sibling element with respect to a reference element by taking a relative selector list as an argument

Functional (An+B) Notation

Represents elements whose numeric position in a series of siblings matches the pattern An+B, for every positive integer or zero value of n. It can be read as the An+B-th element of a list where:

  • A is an integer step size
  • B is an integer offset
  • n is all nonnegative integers, starting from 0

Examples

  • tr:nth-child(odd) or tr:nth-child(2n+1): The odd rows of an HTML table
  • tr:nth-child(even) or tr:nth-child(2n): The even rows of an HTML table
  • nth-child(7): The seventh element
  • nth-child(5n): Every 5th element (5, 10, 15, ...). There is no 0 element, so that doesn't show.
  • nth-child(n+7): The seventh element and each following element (7, 8, 9, ...)
  • nth-child(3n+4): (4, 7, 10, 13, ...)
  • nth-child(-n+3): The first 3 elements (3, 2, 1), but doesn't go beyond 1 because 0 and -n are invalid
  • p:nth-child(n): Every <p> element in a group of siblings. The same as p, but with more specificity.
  • p:nth-child(1) or p:nth-child(0n+1): Every <p> element which is the first one in a group of siblings.
  • p:nth-child(n+8):nth-child(-n+15): the 8th through 15th <p> elements in a series of siblings

Cascading Links Styles (<a>)

The recommended order is :link,:visited,:focus,:hover,:active

The :link and :visited pseudo-classes should generally come first. Next should be :focus and :hover—they're specified now so that they override and apply to both visited and unvisited links. If :focus precedes :hover, the hover effect will apply to links with keyboard input focus. The :active pseudo-class should usually come last, since we usually want to indicate clearly any links that have been activated.

This isn't the only useful order, nor is it in any way the "right" order. The order in which you specify your pseudo-classes will depend on the effects you want to show with different combinations of states

Pseudo-Elements

Style part of a particular element

/* The first line of every <p> element. */
p::first-line {
    color: blue;
}

A CSS pseudo-element is a keyword added to a selector that lets you style a specific part of the selected element(s). For example, ::first-line can be used to change the font of the first line of a paragraph.

Pseudo-Elements
Category Element Description
Text Position ::first-letter

Applies styles to the first letter of the first line of a block-level element, but only when not preceded by other content (such as images or inline tables).

The first letter of an element is not always trivial to identify:

  • Punctuation that precedes or immediately follows the first letter is included in the match. Punctuation includes any Unicode character defined in the open (Ps), close (Pe), initial quote (Pi), final quote (Pf), and other punctuation (Po) classes.
  • Some languages have digraphs that are always capitalized together, like the IJ in Dutch. In these cases, both letters of the digraph should be matched by the ::first-letter pseudo-element.
  • A combination of the ::before pseudo-element and the content property may inject some text at the beginning of the element. In that case, ::first-letter will match the first letter of this generated content.

Only a small subset of CSS properties can be used with ::first-letter. See MDN for the list.

::first-line
