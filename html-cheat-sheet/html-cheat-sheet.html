My HTML/CSS Cheat Sheet

File Tags (Document Structure)

Tag Name Description
<!DOCTYPE> Document Type Tells what version of HTML the document is written in
<html> HTML tag Top-level element of the HTML file (include language "lang" for accessability)
<head> Document Header Holds meta data for the document
<body> Body Wrapper for the main contents of the document

File Tag Attributes

Tag Attribute Description Example
<html> xmlns="http://www.w3.org/1999/xhtml" Specifies xml namespace attribute if you need to conform to XHTML

Header Tags (in <head>)

Tag Name Description
<title> Document Title Shows in the browser tab or title bar
<meta> Meta tags See notes below on meta tags
<script> Script Defines a section for embedded client-side scripts (JavaScript)
<noscript> No Script Define alternate content for sessions that have disabled scripts
<style> Style Defines a section for internal CSS definitions
<link> Link Defines a link to an external style sheet
<base> Base Specifies a default URL and target for all links on the page

Header Tag Attributes (see below for <meta>)

Tag Attribute Description Example
<script> See W3 Schools for attributes & values
<style> See W3 Schools for attributes & values
<link> href="url" defines file to link to
<link> rel="stylesheet" defines file to link to
<link> rel See W3 Schools for more, less common, values
<base> href="url" defines the base url for all relative links on page
<base> target Defines default target for all hyperlinks & forms. See href target for values

Meta Tag Attributes

Attribute/Value Example Description
charset <meta charset="UTF-8"> Character encoding. UTF-8 is the standard
viewport <meta name="viewport" content="width=device-width, initial-scale=1.0"> Renders the page to the width of the device's screen & controls the zoom level
X-UA-Compatible <meta http-equiv="X-UA-Compatible" content="ie=edge"> Specifies the document mode for Internet Explorer. IE=Edge is the highest supported mode.
See W3 Schools for more values

Tag Name Description thead section

Table Tag Attributes

Tag Attribute Description Example

<td> colspan="number" Specifies number of columns to span (default=1) <td colspan="2">
<td> rowspan="number" Specifies number of rows to span (default=1) <td rowspan="2">
<td> headers="header_id" Specifies one or more header cells a row is related to No visual effect. Used by screen readers.

Hyperlink Tags

Tag Name Description Sample
<a> Hyperlink Defines a hyperlink

Hyperlink Attributes (<a>)

Option Description Example Try it
href="url" Defines a link to "url" (site or local file) <a href="http://www.google.com">Google</a> Google
target="_blank|_self|_parent|_top" Indicates where the target page should be opened <a href="http://www.google.com" target="_blank">Google</a> Google
target="_blank" Opens in new tab <a href="http://www.google.com" target="_blank">Google</a> Google
target="_self" Opens in same frame as where clicked (default) <a href="http://www.google.com" target="_self">Google</a> Google
target="_parent" Opens in parent frame <a href="http://www.google.com" target="_parent">Google</a> Google
target="_top" Opens in full body of the window <a href="http://www.google.com" target="_top">Google</a> Google
target="framename" Opens in the named iframe
type="media_type" Specifies the media type of the linked document
media="media_query" Specifies what media/device the linked document is optimized for
download, hreflang, ping, referrerpolicy, rel See W3 Schools for these attributes

Images Tags (in <body>)

Tag Name Description Sample
<img> Image Used to embed an image. Really just a holding place for a referenced image.
<figure> Figure Specifies self-contained content, like illustrations, diagrams, photos, code listings, etc.
<figcaption> Figure Caption defines a caption for a <figure> element. The <figcaption> element can be placed as the first or as the last child of a <figure> element
<picture> Picture Most commonly used to display the "right" image to show based on the viewport See W3 Schools for how this works. Also see <source> below
<svg> SVG Containter Defines a container for SVG graphics
<map> Image Map Defines an image map, which is a series of <area> tags
<area> Image Map Area Defines a clickable area inside an image map
<canvas> Graphics Canvas Defines an area for graphics, to be populated by a script, typically JavaScript

Images Tags Attributes

Tag Attribute Description Example
<img> src="url" (required) Path to the image
<img> alt="text" (required) Alternate text
<img> width="number" Width. Not required, but should be used.
<img> height="number" Height. Not required, but should be used.
<img> srcset="url-list" Gives a URL to a detailed description of the image.
<img> longdesc="url" Specifies a list of images to use in different situations
<img> sizes="sizes" Specifies sizes to use for different page layouts
<img> ismap Specifies an image as a server-side image map
<img> usemap="#mapname" Specifies an image as a client-side image map
<img> loading="eager|lazy" Should image load immediately or until some conditions are met.
<img> referrerpolicy="value" Specifies a referrer policy when fetching an image. See W3 Schools for attributes & values
<img> crossorigin="anonymous|use-credentials" Allows images from third party sites that allow cross-origin access to be used with canvas.

Audio/Video Tags (in <body>)

Tag Name Description Sample
<audio> Audio Embed audio content in a document. Should contain one or more <source> elements
<video> Video Embed video content in a document. Should contain one or more <source> elements
<source> Media Source(s) Define source for <audio>, <video>, or <picture> elements
<track> Text Track Define a text track (close caption) for <audio> and <video> elements

Audio/Video Tags Attributes

Tag Attribute Description Example
<audio> controls (value only) Specifies that audio controls should be shown
<audio> autoplay (value only) Specifies that audio should play when loaded
<audio> loop (value only) Specifies that the audio playback should loop
<audio> muted (value only) Specifies that the audio should be muted
<audio> preload="auto|metadata|none" Gives a hint as to how the file should be loaded
<audio> preload="auto" Prelaod entire file
<audio> preload="metadata" Prelaod only metadata
<audio> preload="none" Do not load file when page loads
<audio> src="url" Gives the source url, but better to use <source> elements
<video> width="number" Width of the video player
<video> length="number" Length of the video player
<video> controls (value only) Specifies that video controls should be shown
<video> controls (value only) Specifies that video controls should be shown
<video> autoplay (value only) Specifies that video should play when loaded
<video> loop (value only) Specifies that the video playback should loop
<video> muted (value only) Specifies that the video should be muted
<video> preload="auto|metadata|none" Gives a hint as to how the file should be loaded
<video> preload="auto" Prelaod entire file
<video> preload="metadata" Prelaod only metadata
<video> preload="none" Do not load file when page loads
<video> poster="url" Specifies an image to show while video is downloading, or until the user hit's "play"
<video> src="url" Gives the source url, but better to use <source> elements
<source> src="url" URL of the media file. Required when used in <audio> or <video> elements
<source> type="MIME-type" Specifies the MIME-type of the resource
<source> media="media_query" Accepts any valid media query normally defined in CSS See W3 Schools for more
<source> srcset="url" Specifies the URL to use in different situations. Required when used in <picture> elements
<source> sizes="string" Specifies image sizes for different page layouts. Works with srcset See dofactory for more info
<track> src="url" Gives the source url for the track file
<track> kind="captions|chapters|descriptions|metadata|subtitles" Specifies the kind of text track
<track> srclang="language_code" Specifies the language of the text file. Required for kind="subtitles"
<track> label="text" Specifies the title of the text track
<track> default (value only) Specifies that the track is to be enables if the user preferences do not indicate that another track would be more appropriate.
