<form> (in <body>)

Tag Name Description Sample
<form> Form Creates an HTML form for user input
The <form> element can contain one or more of the following: <input> <textarea> <button> <select> <option> <optgroup> <fieldset> <label> <output>

<form> Attributes

Tag Attribute Description Notes
<form> name="text" Name of the form, used for JavaScript or to reference data after form is submitted
<form> action="url" Where to send the form-data when submitted
<form> method="get|post" Specifies the http method to use when sending the form-data
<form> method="get" Send form-data as URL variables (free text) Length of URL is limited to about 3000 characters
<form> method="post" Appends form-data inside the body of the HTTP request (not shown in URL)
<form> target Specifies where to display a response. See href target for values
<form> autocomplete="on|off" Specifies autocomplete option. If "on", the browser completes with values user has entered before Can be "on" for form and "off" for some fields, and vice-versa
<form> novalidate Specifies that form should not be validated when submitted
<form> rel Specifies the relationship between the current document and the linked document See W3 Schools for values & descriptions
<form> enctype Specifies how form-data should be encoded. Used only with method="post"
<form> enctype="application/x-www-form-urlencoded" Default. Characters are encoded before sent (modifies spaces & special characters)
<form> enctype="multipart/form-data" Necessary if user will upload a file through the form
<form> enctype="text/plain" No encoding. Not recommended
<form> accept-charset="character_set" A space-separated list of one or more encodings that are used for the form submission. See W3 Schools for character sets reference
